CHAPTER THREE
RESEARCH METHODOLOGY, SYSTEM ANLYSIS AND DESIGN
3.0Introduction
This chapter presents the methodology adopted for the design and implementation of the QR-based cashless transaction system for small-scale businesses using H-Medix Pharmacy and Supermarket, Abuja, as a case study. The methodology explains the various procedures, techniques, tools, and approaches used in carrying out the study and developing the proposed system.
3.1Research Methodology
Research methodology encompasses the various procedures, techniques, methods, and approaches employed in conducting a study. It provides a systematic framework that guides the researcher in achieving the objectives of the research. Methodology is broader than research methods because it includes the principles, strategies, and processes that govern the entire research and development activities. Therefore, research methodology serves as the foundation for planning, data collection, system analysis, design, implementation, testing, evaluation, and documentation of the proposed system.
For this study, a combination of descriptive and developmental research methodologies was adopted to facilitate the design and implementation of the QR-Based Cashless Transaction System for Small-Scale Businesses using H-Medix Pharmacy and Supermarket, Abuja, as a case study. The descriptive methodology was used to examine and understand the existing payment processes, identify their limitations, and determine the requirements needed for an improved solution. The developmental methodology was employed because the study involved the development of a functional software application aimed at solving identified payment-related challenges.
The system was developed using the Software Development Life Cycle (SDLC), specifically the Waterfall Model. The Waterfall Model is a structured and sequential software development approach widely used in information system development. It divides the development process into distinct phases, with each phase completed before proceeding to the next. The major phases include requirement analysis, system design, implementation, testing, deployment, and maintenance. This model was selected because it provides a clear development framework, facilitates proper documentation, and ensures systematic progression throughout the software development process.
The adoption of the Waterfall Model enabled the researcher to analyze the existing payment system, identify operational problems, design the proposed QR-based solution, develop the software, conduct testing procedures, and evaluate the effectiveness of the system before deployment. This structured approach ensured that the developed system met user requirements and provided a secure, reliable, and efficient payment solution for small-scale businesses.
3.2Research Design
The descriptive and developmental research methods were adopted for this study. The descriptive method was used to explain the current payment processes used by small-scale businesses, identify the operational problems associated with existing systems, and describe the features of the proposed QR-based payment solution. This method helped the researcher understand the weaknesses of traditional payment methods such as cash transactions, POS operations, and manual transfer verification.
The developmental research method was adopted because the study involves the design, development, implementation, and testing of a functional software system. The developmental approach provides a systematic framework for developing a solution that addresses identified problems within the existing payment environment. It also supports software planning, system modeling, database design, coding, testing, debugging, and deployment.
The research design followed a systematic procedure consisting of problem formulation, research planning, data collection, data processing, system design, implementation, testing, evaluation, and documentation. This structured approach ensures that the developed system meets user requirements and performs effectively within the selected business environment.
3.3Functional Requirements
The functional requirements describe the specific operations and activities the proposed QR-based cashless transaction system must perform in order to achieve its objectives effectively. These requirements define the major services and features that users will interact with during system operation.
The proposed system should be able to perform the following functions:
User Registration and Login: The system should allow users such as administrators and merchants to register and securely log into the system using usernames and passwords.
QR Code Generation: The system should generate unique QR payment codes containing transaction and payment details for customers.
QR Code Scanning: Customers should be able to scan the generated QR code using mobile banking or payment applications.
Payment Processing: The system should process customer payments instantly after QR code scanning and payment authorization.
Transaction Verification: The system should automatically verify payment status before confirming successful transactions.
Payment Confirmation: The system should display successful payment confirmation messages to both customers and merchants immediately after payment completion.
Transaction Record Storage: The system should store all transaction records securely within the database for future reference and monitoring.
Transaction History Management: The system should allow users to view previous transactions and payment history.
User Management: The administrator should be able to manage users, update records, and monitor transaction activities.
Report Generation: The system should generate transaction reports for business monitoring and financial tracking purposes.
3.3.1Non-Functional Requirements
Non-functional requirements describe the quality attributes, performance standards, and operational characteristics expected from the proposed system. These requirements ensure that the system performs efficiently, securely, and reliably under different operational conditions.
The proposed system should satisfy the following non-functional requirements:
Security: The system should provide secure authentication and protect user information and transaction data from unauthorized access.
Reliability: The system should operate consistently without frequent failures or transaction errors.
Scalability: The system should be capable of handling increasing numbers of users and transaction volumes as business operations expand.
User Friendliness: The system interface should be simple, clear, and easy to navigate for both customers and merchants.
Fast Processing Speed: The system should process transactions quickly in order to reduce customer waiting time and improve operational efficiency.
Accuracy: The system should provide accurate transaction processing and maintain correct financial records without duplication or loss of data.
Availability: The system should be accessible whenever users need to perform transactions or retrieve transaction information.
Maintainability: The system should be easy to maintain, update, and improve whenever necessary.
Compatibility: The system should function properly across different devices such as smartphones, tablets, and computer systems.
Efficiency: The system should utilize system resources effectively while maintaining high transaction performance.
3.4System Architecture
System architecture refers to the structural design that explains how the different components of the QR-based cashless transaction system interact with one another to achieve efficient payment processing and transaction management. The proposed system adopts a three-tier architecture consisting of the presentation layer, application layer, and database layer. This structure was selected because it improves scalability, security, flexibility, and system maintenance.
The presentation layer serves as the user interface where customers, merchants, and administrators interact with the system using smartphones, computers, or web browsers. Through this interface, users can generate QR codes, scan QR codes, make payments, verify transactions, and view transaction history. The interface is designed to be user-friendly and easy to navigate in order to improve customer experience and operational efficiency.
The application layer handles the core business logic of the system. It processes payment requests, verifies transactions, authenticates users, generates QR codes, and communicates with the database layer. This layer acts as the bridge between the user interface and the database. It ensures that all transactions are validated before processing and recording them in the database.
The database layer stores all system information such as user accounts, transaction records, QR code details, payment history, and system logs. MySQL database management system is used because it provides reliability, speed, and efficient data management capabilities. The system architecture therefore ensures secure and efficient communication among all system components.

3.7Entity Relationship Diagram (ERD)
The Entity Relationship Diagram (ERD) describes the relationship between the major entities within the database structure of the QR payment system. It provides a visual representation of how data is stored, connected, and managed within the database environment.
The major entities identified in the system include Users, Transactions, Payments, and QR Codes. The Users entity stores information about customers, merchants, and administrators. The Transactions entity stores payment transaction details such as transaction ID, amount, transaction date, and transaction status. The Payments entity contains payment method details and payment confirmation information, while the QR Codes entity stores generated QR code information linked to transactions.
Relationships exist between these entities to ensure efficient data management. For example, a user can perform multiple transactions, while each transaction generates a corresponding QR code and payment record. These relationships help maintain data integrity and reduce redundancy within the database.
The ERD plays an important role during database implementation because it provides the structural foundation for creating tables, relationships, and constraints in the database management system. It also supports database normalization and improves overall system performance.
By properly organizing the database entities and their relationships, the proposed system achieves efficient storage, easy retrieval of records, and improved transaction management.

The method of data collection refers to the procedures and techniques used by a researcher to obtain relevant information required for the successful completion of a study. Data collection is an important aspect of research because it provides the necessary information for system analysis, design, development, and evaluation. For this study, both primary and secondary sources of data collection were employed to gather information about the existing payment processes and the requirements for the proposed QR-based cashless transaction system.
Primary Source
Primary data refers to original information collected directly from respondents or the study environment for the specific purpose of the research. In this study, primary data were obtained through personal interviews, direct observation, and discussions with the management and staff of H-Medix Pharmacy and Supermarket, Abuja. These methods enabled the researcher to gather firsthand information about the current payment processes, operational challenges, transaction procedures, and user requirements for the proposed system.
Secondary Source
Secondary data refers to information obtained from existing materials that have already been documented by other researchers, organizations, or institutions. The researcher obtained secondary data from textbooks, journals, conference papers, research reports, online publications, articles, and other relevant academic materials related to cashless payment systems, QR code technology, electronic transactions, and software development methodologies. These sources provided theoretical and practical insights that supported the study.
3.9.1Interview Method
The interview method is a systematic process of collecting information through direct interaction between the researcher and respondents. It allows the researcher to obtain detailed explanations, opinions, and experiences regarding the subject under investigation. For this study, interviews were conducted with the management and operational staff of H-Medix Pharmacy and Supermarket, Abuja.
The interview sessions focused on understanding the existing transaction procedures, challenges associated with cash payments, POS transactions, and mobile bank transfers, as well as expectations for an improved payment system. Information obtained from the interviews assisted the researcher in identifying system requirements, operational weaknesses, and user expectations that guided the design and development of the proposed QR-based cashless transaction system.
3.9.2Observation Method
Observation involves the direct examination of activities, processes, and events within the study environment. This method enabled the researcher to observe how payment transactions were processed within the business environment. Through observation, the researcher identified issues such as delayed transaction confirmations, manual verification of transfers, customer waiting time, network-related challenges, and record management practices.
The information gathered through observation complemented data obtained from interviews and provided a clearer understanding of the operational challenges affecting payment processing. The findings from the observation exercise contributed significantly to the design of the proposed QR-based cashless transaction system by ensuring that the developed solution addressed real-life business needs and operational requirements.
3.10Analysis of the Existing System
The existing payment system used by many small-scale businesses mainly involves cash transactions, POS transactions, and mobile bank transfers. Although these payment methods support electronic transactions to some extent, they still experience several operational challenges that reduce efficiency and reliability.
Cash transactions require physical handling of money, which increases the risk of theft, loss, and counting errors. They also make financial record management difficult because transaction records are mostly handled manually. POS systems provide electronic payment services but depend heavily on stable internet connectivity and banking networks. Frequent network failures often lead to unsuccessful or delayed transactions, causing frustration for both customers and business owners.
Mobile bank transfers have also become widely used due to the growth of mobile banking applications. However, this method requires manual verification of payment alerts before goods or services are released. Delayed transaction notifications and fake transfer alerts create operational risks and increase transaction processing time.
The existing system also creates long customer queues during busy periods because payment confirmation sometimes takes several minutes. High POS maintenance costs and transaction charges further increase operational expenses for small businesses. These challenges highlight the need for a simpler, faster, and more reliable digital payment solution such as QR-based payment technology.

3.11Analysis of the Proposed System
The proposed system is a QR-based cashless transaction system developed to improve payment processing and transaction management within small-scale businesses. The system is designed to eliminate many of the operational problems associated with existing payment methods by introducing faster and more reliable digital transaction processing.
The proposed system allows merchants to generate QR payment codes that customers can scan using their mobile banking applications. Once scanned, the payment is processed instantly, and transaction confirmation is automatically sent to both the customer and the merchant. This reduces the need for manual verification and minimizes transaction delays.
The system also maintains digital transaction records within a database, making it easier for business owners to monitor sales activities and generate financial reports. The QR-based system reduces dependence on expensive POS devices and lowers operational costs for small businesses.
In addition, the proposed system improves transaction security through user authentication and automatic payment verification. Since payment details are embedded within QR codes, the possibility of manual entry errors is reduced significantly. The system therefore provides a simple, affordable, secure, and efficient digital payment solution suitable for small-scale retail businesses.
